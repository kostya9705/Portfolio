# Анализ СМС трафика телеком оператора

## Описание проекта

Комплексный анализ временного ряда SMS-трафика телекоммуникационного оператора для выявления закономерностей, обнаружения аномалий и построения прогнозных моделей. Проект включает статистический анализ данных, исследование сезонности и разработку методов детекции нетипичных всплесков нагрузки.

## Структура проекта

## Сравнение методов прогнозирования

**Выбор метрики:** 
- RMSE как основная — штрафует большие ошибки сильнее, что критично для планирования пиковых нагрузок.

| Метод | MSE | RMSE | MAE | Описание |
|-------|-----|---------|-----|----------|
| Среднее по 15-мин интервалам | 51998.15 | 228.03 | 150.79 | Индивидуальное среднее для каждого из 96 интервалов |
| Среднее по периодам суток | 70743.29 | 265.98 | 188.85 | 4 периода: утро, день, вечер, ночь |
| Глобальное среднее (baseline) | 89826.43 | 299.71 | 217.52 | Единое среднее для всех интервалов |

#### Метод 1: Глобальное среднее (baseline)
- Одно значение для всего дня
- Не учитывает сезонность
- Служит точкой отсчёта для сравнения

#### Метод 2: Среднее по периодам суток
- 4 ступенчатых уровня прогноза
- Грубое приближение суточного цикла
- Улучшение на 11% относительно baseline

#### Метод 3: Среднее по 15-минутным интервалам (**Лучший результат**)
- 96 индивидуальных прогнозов
- Точно воспроизводит суточную форму
- Улучшение на 24% относительно baseline
- Стабильно предсказывает нормальное поведение

## Основные выводы

### Структура SMS-трафика
**Суточная сезонность:**
- Минимальная активность: 0-4 часа (220-280 SMS)
- Пиковая нагрузка: 7-15 часов (800-900 SMS)
- Рабочее время формирует 70-75% суточного трафика

**Недельная сезонность:**
- Будние дни: стабильно высокая активность
- Выходные: снижение на 15-20%, особенно утром
- Паттерн повторяется с высокой устойчивостью

**Аномалии:**
- Резкие всплески (>1500 SMS) — потенциальные массовые рассылки
- Провалы трафика — возможные технические сбои
- Частота: 3-5 аномалий в неделю

### Эффективность методов

**Детекция аномалий:**
- Динамические пороги (MA±σ) — точность ~85%, минимум ложных срабатываний
- Статические пороги (µ±σ) — точность ~60%, не учитывает сезонность
- Бизнес-правило — простое, но требует ручной настройки

**Прогнозирование:**
- 15-минутные интервалы — RMSE: 228, улучшение на 24%
- Периоды суток — RMSE: 266, улучшение на 11%
- Глобальное среднее — RMSE: 300, baseline

## Рекомендации для бизнеса

### Мониторинг и алертинг
**Внедрить систему раннего предупреждения:**
- Использовать динамические пороги MA±3σ для критичных алертов
- Настроить уведомления при отклонении >20% от прогноза
- Мониторить как пики, так и провалы трафика

**Приоритеты алертов:**
1. Критичные: провалы трафика >50% (возможные сбои)
2. Высокие: всплески >200% (спам, атаки)
3. Средние: отклонения 50-100% (нестандартные события)
