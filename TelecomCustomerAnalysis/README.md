# Анализ клиентов Телеком оператора

## Описание проекта

Комплексный анализ данных телекоммуникационной компании для выявления факторов оттока клиентов (churn analysis) и построения стратегии удержания. Проект включает исследовательский анализ данных и создание бизнес-правил для сегментации клиентов.

## Структура проекта

```
TelecomCustomerAnalysis/
├── analysis.ipynb          # Jupyter notebook с полным анализом
├── requirements.txt        # Зависимости проекта
├── data/
│   └── telco_customer_data.csv
└── README.md              
```

## Цели анализа

1. Определить основные характеристики клиентов, склонных к оттоку
2. Сегментировать пользователей по ключевым признакам
3. Найти закономерности для выстраивания стратегии удержания
4. Провести визуализацию распределений и их влияния на отток

## Структура данных

**Группы признаков:**
- **Демографические:** Age, Gender, Married, Dependents
- **География:** City, State, Country, Zip Code
- **Контракт:** Contract, Tenure in Months
- **Финансовые:** Monthly Charge, Total Revenue, CLTV
- **Услуги:** Internet Service, Streaming TV, Premium Tech Support
- **Целевая переменная:** Churn (факт оттока)

## Ключевые этапы анализа

### 1. Предварительная обработка
- Удаление признаков с >50% пропусков (Churn Category, Churn Reason, Offer)
- Заполнение пропусков модой для Internet Type
- Проверка на дубликаты

### 2. Корреляционный анализ
Применены различные методы для каждого типа признаков:

**Числовые признаки** (Point-Biserial Correlation):
- **Положительная связь:** Monthly Charge (+0.18), Age, Population
- **Отрицательная связь:** Tenure in Months, Total Revenue, Number of Referrals

**Категориальные признаки** (Cramér's V):
- **Сильная связь:** Satisfaction Score, Contract, Zip Code
- **Слабая связь:** Internet Type, City

**Бинарные признаки** (Phi-коэффициент):
- **Значимые:** Dependents, Internet Service, Paperless Billing
- **Незначимые:** Phone Service, Gender, Multiple Lines

### 3. Сегментация клиентов

**По возрасту:**
- ≤20 лет: ~200 клиентов (молодёжная аудитория)
- 21-65 лет: >3000 клиентов (основная масса)
- 65+ лет: ~600 клиентов (пожилые пользователи)

**По риску оттока (Churn Score):**
- Низкий: <400 клиентов
- Средний: ~1500 клиентов
- Высокий: ~1800 клиентов
- Очень высокий: ~700 клиентов

## Основные выводы

### Факторы, увеличивающие отток:
1. **Тип контракта Month-to-Month** — самый высокий churn (~45%)
2. **Высокие ежемесячные платежи** — клиенты чувствительны к цене
3. **Отсутствие дополнительных услуг** — Online Security, Tech Support
4. **Низкая удовлетворённость** (Satisfaction Score)

### Факторы, снижающие отток:
1. **Долгосрочные контракты** (One Year, Two Year) — churn до 3%
2. **Реферальная программа** — клиенты по рекомендациям более лояльны
3. **Длительность обслуживания** (Tenure) — стабильность растёт со временем
4. **Высокий CLTV** — ценные клиенты реже уходят

## Бизнес-правило для предсказания оттока

Разработано простое правило на основе среднего месячного платежа:
- Клиенты с платежом ≥ среднего у ушедших → риск оттока

**Метрики:**
- **Accuracy:** 59.1%
- **Precision:** 34.6% (много ложных срабатываний)
- **Recall:** 60.9% (обнаруживает большинство уходящих)

> Правило приемлемо для первичной сегментации, но требует улучшения через ML-модели

## Рекомендации для бизнеса

1. **Программы лояльности:**
   - Развивать реферальные программы
   - Создать накопительную систему бонусов

2. **Удержание клиентов:**
   - Стимулировать переход на долгосрочные контракты
   - Предлагать персонализированные тарифы для клиентов с высоким Monthly Charge

3. **Продуктовая стратегия:**
   - Включать защитные сервисы в базовые тарифы
   - Создать семейные пакеты услуг

4. **Работа с риском:**
   - Приоритизировать клиентов с высоким Churn Score
   - Проактивные кампании для удержания перед окончанием контракта

5. **Сегментированный подход:**
   - Молодёжь (<25): гибкие digital-тарифы
   - Пожилые (>65): упрощённые тарифы с премиум-поддержкой
